\section{Recursos sobre el proyecto}

\subsection{Página web}

Una vez sea presentado el proyecto mi intención es seguir desarrollando
MIEX en mis ratos libres. Para hacer el trabajo más transparente, existe un
proyecto creado en SourceForge\footnote{http://www.sourceforge.net} así como una
página web en la siguiente dirección.

\url{http://miex.sf.net}

A través de esa URI es posible obtener la información más actualizada acerca de
MIEX, estar a día de los eventos que rodean al proyecto y obtener
\textit{snapshots} actualizadas del código fuente.

\subsection{RSS}

Si desea suscribirse al canal RSS\footnote{http://es.wikipedia.org/wiki/RSS} que
contiene las noticias no tiene más que añadir la siguiente dirección en su
agregador favorito:

\url{http://sourceforge.net/export/rss2_projnews.php?group_id=187602}

\subsection{Subversion}

El repositorio donde se lleva el control de las versiones del código fuente
(utilizando Subversion\footnote{http://subversion.tigris.org/}) se
encuentra en \url{https://miex.svn.sourceforge.net/svnroot/miex} y tiene la
siguiente estructura:

\begin{itemize}
	\item \textbf{branches} - Ramas de desarrollo paralelas a
\textit{trunk}
	\item \textbf{tags} - Versiones congeladas (releases)
	\item \textbf{trunk} - Rama principal de desarrollo de MIEX 
	\begin{itemize}
		\item \textbf{doc} - Documentación (presentaciones, libros...)
		\item \textbf{miex} - Código fuente del proyecto
		\item \textbf{web} - Página web
	\end{itemize}
\end{itemize}

\textbf{Nota:} Use el contenido de \textit{trunk} con precaución. Debido a que
es código que suele cambiar con mucha frecuencia, puede obtener una captura
inestable y obtener resultados inesperados. Si desea una versión estable y
funcional descárgela directamente de la página web o obténgala desde
\textit{tags}.
