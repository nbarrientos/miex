\section{Los ficheros de configuración}

MIEX utiliza un fichero de configuración de tipo \textit{clave=valor} para
personalizar el comportamiento del programa. El fichero de configuración de
ejemplo que se distribuye con el software está bastante explicado, por lo que en
esta sección simplemente se hará referencia a él.

Las opciones de configuración existentes hasta la fecha son las siguientes:

\begin{Verbatim}[frame=single,framesep=2pt,fontsize=\relsize{-1}]
# DBHostname
#   Type:     String
#   Default:  127.0.0.1
#   Desc:     Self-explaining.
#   See also: DB*

# DBPort
#   Type:     String
#   Default:  3306
#   Desc:     Self-explaining.
#   See also: DB*

# DBUser
#   Type:     String
#   Default:  - (Required field)
#   Desc:     Self-explaining.
#   See also: DB*

# DBPassword
#   Type:     String
#   Default:  - (Required field)
#   Desc:     Self-explaining.
#   See also: DB*

# DBName
#   Type:     String
#   Default:  - (Required field)
#   Desc:     Self-explaining.
#   See also: DB*

# Validate
#   Type:     Yes/No (boolean)
#   Default:  No
#   Desc:     Tries to validate each collection using the provided Schema.
#   See also: XMLschemaURI

# XMLschemaURI
#   Type:     String
#   Default:  /usr/share/miex/schemas/default.xsd
#   Desc:     Path to a XML Schema used later to validate input files.
#   See also: Validate

# GrammarURI
#   Type:     String
#   Default:  /usr/share/miex/grammars/englishPCFG.ser.gz
#   Desc:     Path to a grammar file, usually shipped along with 
#             the Stanford Parser.
#   See also: -

# Dump
#   Type:     Yes/No (boolean)
#   Default:  No
#   Desc:     Creates one file per category containing its respective 
#             sentences.
#   See also: DumpDir, SkipProcessing

# DumpDir
#   Type:     String
#   Default:  /tmp
#   Desc:     Path to a directory where the files will be dumped.
#   See also: Dump

# SkipProcessing
#   Type:     Yes/No (boolean)
#   Default:  No
#   Desc:     Stops the processing after create the per-category files.
#             (This setting only would make sense whether Dump is set 
#             to Yes).
#   See also: Dump, DumpDir

# CreateDB
#   Type:     Yes/No (boolean)
#   Default:  No
#   Desc:     Creates the DB from scratch before start processing the 
#             input. USE CAREFULLY.
#   See also: SQLSkeleton

# SQLSkeleton
#   Type:     String
#   Default:  /usr/share/miex/sql/skeleton.sql
#   Desc:     Path to a text file containing the statements to create 
#             an empty DB.
#   See also: CreateDB

# Normalize
#   Type:     Yes/No (boolean)
#   Default:  No
#   Desc:     Activates Porter support and saves normalized words as well.
#   See also: -
\end{Verbatim}

Una vez instalado el programa como se indica en el siguiente capítulo, se puede
tomar como plantilla el fichero de ejemplo que se encuentra en el directorio:

\texttt{\$PREFIX/share/doc/miex/examples/conf}
