\section{Otras utilidades}

\subsection{Scripts de transformación}

Aunque estos scripts tienen una funcionalidad muy concreta y han sido diseñados
para un caso de uso muy especial, se distribuyen igualmente por si resultaran
necesarios. \textbf{No los utilice si no sabe bien lo que está haciendo.}

Durante el periodo de pruebas se detectó la existencia de algunos ficheros de
entrada que no estaban bien formados, por lo que el analizador no era capaz
de tratarlos correctamente. El script llamado \textbf{\texttt{fixxml.sh}},
situado en el directorio \texttt{misc/transformation} y dependiente de
\textbf{gawk} toma como argumento la ruta de un fichero a arreglar e imprime el
fichero correcto por la salida estándar.

Los errores que éste script arregla son los siguientes:

\begin{itemize}
\item Añade un nodo raíz al fichero de colección (\texttt{<collection>} y
\texttt{</collection>})

\item Finaliza correctamente las etiquetas de las secciones
\texttt{doctrain} y \texttt{doctest} de ficheros de colección erróneos
(por ejemplo, los que comiencen por \texttt{<doctrain>} y acaben con
\texttt{</doc>}, se cambiaría esta última etiqueta por \texttt{</doctrain>})
\end{itemize}

Recuerde, MIEX \textbf{excluirá del procesamiento} a los ficheros que no cumplan
el \textit{schema} proporcionado. Si usted fuerza evitar la
comprobación utilizando \textit{schema} y el fichero de colección está mal
formado, el \textit{unmarshaller} lanzará una excepción no recuperable y
abortará la ejecución.

Puede obtener más información acerca del formato correcto de los ficheros de
entrada en la sección \ref{sec:schema} de este mismo libro.

\subsection{Integración con FK}

FK\footnote{http://www.aic.uniovi.es/pir/Software.htm} es un entorno para el
desarrollo de aplicaciones modulares en  C++ basado en un modelo de plugins
jerárquicos, extensibles y reutilizables. FK\cite{FK} ofrece al programador
servicios para cargar dinámicamente plugins que tengan un cierto interfaz y
automáticamente comprueba, convierte y enlaza las instancias particulares
especificadas en tiempo de ejecución. Este programa forma parte de un proyecto
fin de carrera desarrollado por Manuel Roberto Berdasco Martínez.

Como herramienta experimental, en el directorio \texttt{misc/fk} se distribuye
un \textit{wrapper} para poder invocar fácilmente MIEX (y por consiguiente la
JVM\footnote{Java Virtual Machine})
desde FK.

Hasta el momento no existe ninguna manera de configurar el \textit{wrapper} en
tiempo de ejecución por lo que probablemente tendrá que modificar algún
\textit{path} en el fichero fuente antes de compilar y ejecutar para que
funcione correctamente.

Finalmente, el autor de esta documentación \textbf{no recomienda} lanzar MIEX
utilizando FK, utilice el sistema habitual para realizar las ejecuciones.
