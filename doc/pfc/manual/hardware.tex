\section{Requisitos hardware}

Como se advierte en la documentación del Stanford Parser el coste computacional
del análisis es muy alto, por lo que es recomendable disponer de una máquina de
prestaciones medias/altas para obtener resultados en un tiempo aceptable.

Por falta obvia de recursos, no se ha podido hacer pruebas en diferentes
configuraciones hardware por lo que, exceptuando el sistema operativo, tómense
los siguientes requisitos como una recomendación.

\begin{itemize}

\item \underline{Arquitectura}: Cualquier arquitectura es válida,
mientras exista una máquina virtual de Java para ella (de Sun Microsystems o
compatible con la versión establecida en los requisitos software) y un sistema
operativo (recomendado GNU/Linux) que la soporte. Una elección aceptable sería
tomar una de las más populares, por ejemplo x86-32 (IA-32) o x86-64 (EMT64).

\item \underline{Procesador}: Cualquier procesador que implemente una de las
arquitecturas válidas. La frecuencia de funcionamiento y la cantidad de caché,
dentro de unos márgenes lógicos, a convenir.

\item \underline{Memoria}: El consumo de memoria por parte del análisis de las
oraciones es bastante elevado, por lo que para no tener que hacer demasiado uso
de la memoria de intercambio, se recomiendan cantidades de memoria física
iguales o superiores a 1 GiB.

\item \underline{Disco}: A convenir (si va a instalar la base de datos en el
equipo local, evalue el número de colecciones a analizar para elegir una
capacidad adecuada).

\item \underline{Sistema Operativo}: Se recomienda cualquier distribución de
\textbf{GNU/Linux}, aunque cualquier sistema operativo sobre el se pueda correr
una máquina virtual de Java podría servir.

\item \underline{Otros}: A convenir.

\end{itemize}
